<template> Request Demo

Example apps that use a template protocol for modern web UX.

What is a template request?

An web request that results in a HTML template being applied to update the loaded webpage.

Motivation

The goal is for conventional web servers to support basic interactivity. This will make modern web apps less reliant on custom APIs for simple dynamic controls.

The protocol

An Accept[1] header is used to indicate to the web server that the client is expecting a template as a response. If implemented, a HTML template fragment will be sent in the response body. That template will be applied to the requesting document using a find and replace strategy. 

> GET /content_a HTTP/1.1
  Accept: application/x.treetop-html-template+xml

< HTTP/1.1 200 OK
  Content-Type: application/x.treetop-html-template+xml
  X-Page-URL: /content_a
  Vary: Accept

  <template>
    <div id="content">...</div>
    <div id="nav">...</div>
  </template>

That might seems too simple to be useful so these example apps were created to demonstrate how various controls can be implemented using the protocol.

Third-party libraries

As you might have already guessed, this page and all demo apps use the Bootstrap Component Library. Styling and markup is intended to be as familiar and conventional as possible[2].

The protocol described here was developed along with the Treetop library, a Go package for building HTML web pages from nested templates. That library is used to implement the server-side handlers for the demo apps.
