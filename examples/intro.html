<template/> Request Demo

A series of example apps that demonstrate the use of a template protocol to support modern web UX.

What is a template request?

A convention that web links, forms and components can use, with XHR, to send a normal request and receive an update for the current page instead of a new HTML document.

Motivation

The goal is to allow a standard HTML web server to support basic interactivity by extending the protocol that is already in use. This will hopefully reduce the need for web apps to expand custom APIs to support minor dynamic controls.

The protocol

Using an Accept[1] header, the client indicates to the web server that a template is expected as a response. If implemented, a HTML template fragment will be sent in the response body. That template will be applied to the requesting document using a find and replace strategy. 

> GET /content_a HTTP/1.1
  Accept: application/x.treetop-html-template+xml

< HTTP/1.1 200 OK
  Content-Type: application/x.treetop-html-template+xml
  X-Page-URL: /content_a
  Vary: Accept

  <template>
    <div id="content">...</div>
    <div id="nav">...</div>
  </template>

That might seems too simple to be useful so the example apps were created to demonstrate how common controls can be implemented using the protocol.

Third-party libraries

As you might have already guessed, this page and all demo apps use the Bootstrap Component Library. Styling and markup is intended to be as familiar and conventional as possible[2].

The protocol described here was developed along with the Treetop library, a Go package for building HTML web pages from nested templates. That library is used to implement the server-side handlers for the demo apps.
